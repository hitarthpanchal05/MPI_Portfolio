01 — Introduction

Project Overview

Water is one of our most critical resources, and its efficient management is essential in both industrial and domestic settings. Uncontrolled overflow from tanks wastes thousands of litres daily and can cause structural damage, flooding, and energy wastage. This project addresses these challenges with an intelligent, embedded real-time monitoring and alert system.

Using the STM32 NUCLEO-L476RG 32-bit ARM Cortex-M4 microcontroller and the HC-SR04 ultrasonic distance sensor, the system continuously measures the air-gap above the water surface. The level percentage is computed and streamed to the STM32CubeIDE SWV ITM Data Console over SWD debug interface. When the water rises to or beyond 80% of tank capacity, an external LED and active buzzer are activated, delivering immediate visual and audible warnings.

📡

Ultrasonic Sensing

HC-SR04 emits 40 kHz pulses and measures echo return time to derive precise distance with ±3 mm accuracy over 2–400 cm range.

🖥️

SWV ITM Console

printf-style output routed through ARM ITM stimulus port 0 over SWD wire, displaying live water level readings without UART overhead.

🚨

Threshold Alert

Configurable 80% fill threshold triggers simultaneous LED flash and buzzer tone—immediate multisensory notification of approaching overflow.

⚡

Low Power ARM Core

L476RG's ultra-low-power modes and 80 MHz Cortex-M4 ensure accurate microsecond timing for ultrasonic pulse measurement.

04 — Circuit Diagram

Schematic & Connections

The HC-SR04 is powered from 5V and its Trigger/Echo lines connect to STM32 GPIO pins. The LED and buzzer share the same alert GPIO, with the buzzer driven through an NPN transistor to protect MCU output current limits.

STM32 ↔ HC-SR04 Connections

5V Pin→ HC-SR04 VCC
GND→ HC-SR04 GND
PA0→ HC-SR04 TRIG (Output)
PA1← HC-SR04 ECHO (Input)

STM32 ↔ Alert Components

PA5→ LED Anode (via 220Ω)
PA6→ NPN Base (via 1kΩ) → Buzzer
GND→ LED Cathode / Transistor Emitter
3.3V/5V→ Buzzer + (via Collector)
05 — Configuration & Code

STM32CubeIDE Setup & Firmware

The project uses STM32CubeMX code generation within STM32CubeIDE. TIM2 is configured in input capture / PWM mode at 1 MHz (1 µs resolution) for precise echo pulse timing. PA0 is configured as GPIO output (trigger), PA1 as GPIO input (echo), PA5 and PA6 as GPIO outputs for LED and buzzer respectively. The SWV ITM printf retarget routes printf to ITM stimulus port 0 at a core clock of 80 MHz.

CubeMX Configuration Summary

TIM2 (Timing)

PSC = 79, ARR = 65535 → 1 µs tick. Used for 10 µs TRIG pulse and echo duration measurement with microsecond precision.

GPIO Setup

PA0: Output PP (TRIG). PA1: Input (ECHO). PA5: Output PP (LED). PA6: Output PP (Buzzer via NPN). All 3.3V logic.

SWV / ITM

Enable SWO in Debug config. Core clock = 80 MHz. SWV clock = 4 MHz. Add ITM_SendChar retarget for printf to ITM port 0.

main.c — Core Application Code

main.c
/* Water Level Monitoring & Alert System
 * STM32 NUCLEO-L476RG + HC-SR04
 * Author: Hitarth Panchal | USN: 01FE25BAR702
 * Branch: Automation & Robotics | Sem 4
 */

#include "main.h"
#include <stdio.h>
#include <stdint.h>

/* ---- Configuration -------------------------------------------------- */
#define TANK_HEIGHT_CM   20.0f   /* total tank height in cm    */
#define ALERT_THRESHOLD  80.0f   /* alert at 80% fill          */
#define SOUND_SPEED      0.0343f /* cm/µs at 20°C              */
#define DELAY_MS         500     /* measurement interval (ms)  */

/* ---- ITM printf retarget -------------------------------------------- */
int __io_putchar(int ch) {
    ITM_SendChar((uint32_t)ch);
    return ch;
}

/* ---- Microsecond delay using TIM2 ----------------------------------- */
void delay_us(uint32_t us) {
    __HAL_TIM_SET_COUNTER(&htim2, 0);
    while (__HAL_TIM_GET_COUNTER(&htim2) < us);
}

/* ---- Send 10µs trigger pulse to HC-SR04 ----------------------------- */
void HC_SR04_Trigger(void) {
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_0, GPIO_PIN_SET);
    delay_us(10);
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_0, GPIO_PIN_RESET);
}

/* ---- Measure echo pulse duration in µs ------------------------------ */
uint32_t HC_SR04_GetEchoUs(void) {
    uint32_t t_start = 0, t_end = 0;
    uint32_t timeout = 30000;   /* 30 ms max wait */

    /* Wait for ECHO HIGH */
    __HAL_TIM_SET_COUNTER(&htim2, 0);
    while (!HAL_GPIO_ReadPin(GPIOA, GPIO_PIN_1)) {
        if (__HAL_TIM_GET_COUNTER(&htim2) > timeout) return 0;
    }
    t_start = __HAL_TIM_GET_COUNTER(&htim2);

    /* Wait for ECHO LOW */
    while (HAL_GPIO_ReadPin(GPIOA, GPIO_PIN_1)) {
        if (__HAL_TIM_GET_COUNTER(&htim2) > timeout) return 0;
    }
    t_end = __HAL_TIM_GET_COUNTER(&htim2);

    return (t_end - t_start);
}

/* ---- Calculate water level percentage ------------------------------- */
float getWaterLevelPercent(float distanceCm) {
    float waterHeight = TANK_HEIGHT_CM - distanceCm;
    if (waterHeight < 0) waterHeight = 0;
    float pct = (waterHeight / TANK_HEIGHT_CM) * 100.0f;
    return (pct > 100.0f) ? 100.0f : pct;
}

/* ---- Set alert peripherals (LED + Buzzer) --------------------------- */
void setAlert(uint8_t enable) {
    GPIO_PinState s = enable ? GPIO_PIN_SET : GPIO_PIN_RESET;
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, s);  /* LED   */
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_6, s);  /* Buzzer */
}

/* ---- Main loop ------------------------------------------------------ */
int main(void) {
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    MX_TIM2_Init();

    HAL_TIM_Base_Start(&htim2);

    printf("\r\n=== Water Level Monitor | Hitarth Panchal ===\r\n");
    printf("Tank Height : %.1f cm | Alert at : %.0f%%\r\n\n",
           TANK_HEIGHT_CM, ALERT_THRESHOLD);

    while (1) {
        HC_SR04_Trigger();

        uint32_t echoUs   = HC_SR04_GetEchoUs();
        float    distCm   = (echoUs * SOUND_SPEED) / 2.0f;
        float    levelPct = getWaterLevelPercent(distCm);

        printf("Distance: %6.2f cm | Level: %5.1f%%", distCm, levelPct);

        if (levelPct >= ALERT_THRESHOLD) {
            setAlert(1);
            printf("  [!] ALERT — TANK OVERFLOW RISK\r\n");
        } else {
            setAlert(0);
            printf("  [ ] Normal\r\n");
        }

        HAL_Delay(DELAY_MS);
    }
}

Program Flow

START — SYSTEM INIT
HAL / GPIO / TIM2 Init
1 µs tick; ITM printf ready
Send 10µs TRIG Pulse → HC-SR04
Measure ECHO pulse duration (µs)
Compute distance: d = (t × 0.0343) / 2
Calculate level% = ((H − d) / H) × 100
Print to SWV ITM Console
Level ≥ 80% ?
YES → Activate LED + Buzzer
Delay 500ms → Loop
REPEAT ↺

09 — Conclusion

Summary & Outcomes

This experiment successfully demonstrates a fully functional, real-time Water Level Monitoring and Alert System built on the STM32 NUCLEO-L476RG microcontroller. The HC-SR04 ultrasonic sensor provides reliable non-contact distance measurements with ±3 mm accuracy, and the firmware converts these into water level percentages displayed live on the SWV ITM Data Console.

The 80% threshold-triggered alert system — combining an external LED and active buzzer — provides immediate multisensory notification, successfully preventing tank overflow during all test scenarios. The use of STM32's TIM2 peripheral at 1 µs resolution ensures accurate echo timing critical to the system's measurement precision.

From an academic perspective, this project integrates GPIO control, timer configuration, ARM SWV debugging, C firmware development, and analog-to-digital sensor interfacing — forming a comprehensive foundation in embedded systems engineering relevant to the Automation and Robotics discipline. The modular code architecture is readily extensible to include UART logging, I2C LCD display, or MQTT IoT integration.
